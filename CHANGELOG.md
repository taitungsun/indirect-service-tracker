# Changelog

本專案的版本更新紀錄。

## [1.2] - 2026-07

### 新增 (Added)

- **網站圖示 (Favicon)**：以內嵌 SVG (data URI) 方式加入，維持單一 HTML 檔、離線可用的特性，不需額外的 `.ico` 圖檔。圖示為系統主色藍底的剪貼板紀錄圖案，瀏覽器分頁與書籤皆會顯示。

### 修正 (Fixed)

- **「日期」欄位預設值的時區錯誤**：原本使用 `valueAsDate = new Date()` 設定預設日期，該屬性是以「UTC 日期」解讀，台灣 (UTC+8) 在**每天早上 8 點以前開啟系統時，預設日期會誤顯示成「昨天」**，早上到校即登錄紀錄的老師很容易誤植日期。現改為以本地時區組出 `YYYY-MM-DD` 字串直接指定，任何時間開啟都正確顯示今天。（影響「頁面載入」與「儲存後重置表單」兩處，皆已修正。）
- **服務對象統計失真**：成果報告中，服務對象為「無 (非諮詢服務)」的紀錄原本會被併入「其他」人次計算，造成「其他」對象人次虛增、與實際不符。現在「無」獨立成一列統計，總數仍與紀錄筆數一致。

### 變更 (Changed)

- 「還原 JSON」的檔案選擇器加上 `accept=".json"` 限制，避免誤選其他格式的檔案。

### 安全性與穩健性 (Security & Robustness)

- **編輯內容遺失防呆**：編輯紀錄按下儲存時，若該筆紀錄已不存在（例如同時開啟兩個分頁、已在另一個分頁被刪除），原本會靜默失敗、修改內容直接遺失。現在會偵測此狀況並提示「找不到原紀錄」，改以新增方式保留使用者輸入的內容。

## [1.1] - 2026-06

### 修正 (Fixed)

- **列印時統計圓餅圖覆蓋表格內容**：原因是 Chart.js 依「螢幕」版面計算 canvas 尺寸，但瀏覽器切換到「列印」版面 (`@media print`) 的時機與圖表重新計算尺寸的時機沒有同步，導致 canvas 維持螢幕尺寸而溢出容器、蓋住下方表格。現改為監聽 `beforeprint`/`afterprint` 事件，在列印樣式確實套用後才重新計算圖表尺寸，並對圖表容器加上 `overflow:hidden` 作為雙重防護。
- **項目統計總數兜不起來**：當紀錄的主要項目不在五大分類中時（例如還原了舊版或外部資料），原本會被直接忽略、不計入任何統計，導致報表總數與實際紀錄數不一致。現已併入「其他」類別計算，邏輯與服務對象統計一致。
- **列印時無法選擇「直式」**：原本紙張方向是系統自動判斷，只要勾選列印圖表就會強制橫式，使用者無法手動覆寫。新增列印選項視窗中的「紙張方向」選擇（自動 / 直式 / 橫式），可自由指定。
- **列印時圖表文字對比度不佳**：圖表文字顏色原本依「畫面目前的深色模式設定」決定，但紙張永遠是白色，深色模式下直接列印會讓淺色文字幾乎看不見。現在列印時會將圖表的文字與格線顏色強制改為淺底深字的「列印安全配色」，列印結束後再還原成原本的主題設定。
- **列印時圖表超出紙張寬度被裁切**：原本「統計表格＋長條圖」左右並排各佔一半寬度，圖表寬度的計算時機若與列印版面實際生效的時機沒有完全同步，算出來的寬度可能偏大、超出紙張右側被裁掉。改為列印時「表格在上、圖表在下」垂直堆疊，不再需要左右平分寬度；並加上 `max-width:100%` 防護，確保即使計算有誤也只會等比例縮小，不會遺失內容。
- **列印時圖表高度被裁切**：長條圖數值較小的長條會整根消失、Y 軸不是從 0 開始，圓餅圖上緣被裁成一個平邊。原因是量到的圖表高度是切換列印模式前、畫面上的舊高度，比列印實際容器還高，超出部分被裁掉看不到。改為列印時明確指定圖表高度，並加上 `max-height:100%` 防護，雙重確保不再發生資料被裁切看不到的狀況。

### 新增 (Added)

- 列印選項新增「紙張方向」選擇（自動 / 直式 / 橫式）。
- 還原 JSON 備份、或資料本身缺漏欄位時，新增資料正規化機制 (`normalizeRecord`)，統一補上預設值與型別檢查，避免畫面因資料格式異常而整個壞掉。

### 變更 (Changed)

- 成果報告版面順序調整：將「分散式資源班間接服務紀錄詳表」（文字資料）移到「間接服務項目統計分析」「服務對象統計分析」（統計圖表）之前，畫面顯示與列印順序皆為先看明細、再看統計圖表。
- 圓餅圖標籤優化：切片佔比過小（< 8%）時不再顯示文字標籤，避免與相鄰小切片互相重疊、難以閱讀；精確數字仍可從旁邊的統計表格與圖例讀取。標籤文字顏色改為依切片底色動態計算對比色（例如黃色切片會自動改用黑字而非白字）。

### 安全性與穩健性 (Security & Robustness)

- 匯出 Excel (CSV) 時，所有欄位皆正確處理逗號、引號、換行，避免欄位內容含特殊符號時跑欄位錯位。
- 學生姓名、服務摘要等使用者輸入文字，輸出至畫面前皆會進行 HTML 轉義，避免特殊符號（如 `<` `>` `&`）造成版面跑掉或潛在的注入風險。
- 簽名表格（資源班教師／單位主管）原本只靠 inline style 控制顯示，列印過一次「詳細紀錄表」後會永久殘留在畫面上；改為 CSS 預設隱藏，僅列印時依設定顯示。
- 讀取本機資料 (`localStorage`) 時加上例外處理，避免資料毀損時導致整個系統載入失敗。

## [1.0] - 2026-01

- 初始版本發布。
- 單一 HTML 檔案架構，免安裝、跨平台、資料落地儲存。
- 紀錄管理與儀表板、列印報表、Excel (CSV) 匯出、JSON 備份與還原、深色模式切換。
- 已知問題：列印報表時，統計圓餅圖在某些解析度下可能覆蓋部分表格內容（已於 V1.1 修正）。
